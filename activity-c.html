💉 SQL Injection

Most websites store data in a database using SQL. When developers forget to sanitize user input, attackers can inject SQL commands directly into queries — tricking the database into bypassing authentication entirely.

🎯 CHALLENGE: Log in without knowing the password

You don't know the password — but that won't matter. The SQL injection will make the condition always true, bypassing the check entirely.

The database will evaluate your injected query. Because 1=1 is always true, the WHERE condition passes and you're logged in — no password needed.
💡
The -- in SQL means "comment everything after this". So admin' -- closes the username string and comments out the password check entirely. The query becomes: SELECT * FROM users WHERE username = 'admin' — no password required!

🧠 What Just Happened?

The server built the SQL query by directly concatenating your input without checking it. When you typed ' OR 1=1 --, the quote character broke out of the string, and your SQL became part of the actual database command.

The fix is simple: use Prepared Statements (also called parameterized queries). These separate the SQL code from the data, so no matter what a user types, it's treated as data — never as executable SQL.

SQL injection has been the #1 web vulnerability for over a decade and caused breaches at Yahoo, LinkedIn, Sony, and many more.
