🏦 The Bank Heist

Every website you visit is just HTML, CSS, and JavaScript — and you can edit any of it live in your browser. In this activity, you'll use DevTools to change the price of a laptop on a fake store... and "buy" it for $1.

DevTools will highlight the element. You'll see something like <div class="product-price" id="price">$4,999.00</div> in blue.

Double-click on the $4,999.00 text inside the HTML tag. Type $1 and press Enter. Watch the price on the page change!

Close DevTools and click the Buy Now button. Congrats — you just got a MacBook Pro for $1! 🎉

Shortcut: On Windows, press F12 to open DevTools instantly. On Mac, press Cmd + Option + I. Then use the Elements tab and look for the element with id="price".

🧠 What Just Happened?

You edited the frontend of the website — the part that only exists in your browser. The change is visual only; a real secure server would verify the actual price before processing payment.

But here's the scary part: some poorly coded websites DO trust the frontend. Early e-commerce sites would send the price from the browser to the server — meaning attackers could manipulate it exactly like this and pay less.

This is why developers must never trust client-side data. All pricing, permissions, and critical logic must live on the server, not the browser.

Teaching point: a website usually has two big parts working together: the frontend and the backend.

Frontend

The frontend is everything users see and interact with in the browser, like text, buttons, images, forms, and menus.

Backend

The backend runs on the server. It stores data, checks permissions, processes requests, and decides what is actually allowed.

HTML

HTML gives a webpage its structure. It creates things like headings, paragraphs, links, buttons, and images.

CSS

CSS controls how the page looks, including colors, fonts, spacing, borders, and layout.

JavaScript

JavaScript adds behavior and interactivity. It can react to clicks, update content, and change what appears on screen.

This Activity

In this challenge, you changed the frontend in your own browser. A secure backend should still check the real price before accepting payment.
